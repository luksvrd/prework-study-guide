HTML

  • The head element contains information or metatdata helping the browser render the webpage.
    • These elements are not visible to a user.
  • The body element represents the visible content shown to the user.
  • "ul" starts a list in bullet point format
    • followed by "li" (list items) preceded with a solid dot called the list item marker
    • List item markers can be changed using CSS
  • "ol" is similar to "ul" except it creates a list in numbered format.
    • followed by "li" list items.
    • ordered lists have unique attributes availible such as start and reverse, by default ol's starts at 1
  • "li" elements have to be a direct child to either the "ul" or "ol" parent element.
  • A section tag with an "h2" was used to created the separate HTML, CSS, Git, and JavaScript sections.

CSS

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • You must link your styles.css file to the index.html file for the changes to work
  • Use an external CSS Style Sheet and follow the CSS box model.

Git

  1. git status: checks what branch we are currently on
  2. git checkout -b branch-name: creates a new branch and switches to it
  3. git add -A: once you've edited and saved a file to stage the changes.
  4. git commit -m "changes described": changes are added and commited locally
  5. git pull origin main: pulls in the base branch (main) to your feature branch
  6. git push arigin feature/add-file: pushes your changes to the remote repository. Always push to your feature branch instead of main to keep bugs out.

JavaScript

  • Variable: (var) container used to store data in your code.
  • Comment out lines of code with quick key (command) (/). This keeps the code but doesn't execute it.
  • JS changes can be seen by right clicking on a webpage and selecting "Inspect" - used for debugging and testing code.
  • You must link the .js file to the index.html file for the changes to work.
